Hexo 搭建与入门指南

Hexo 简介

Hexo 是一个基于 Node.js 的强大、简洁的静态博客框架。支持快速生成静态网页，可以充分利用CDN加速，提供高速访问体验。像 Typecho 一样，Hexo 支持 Markdown，写作体验流畅。

Hexo 提供了多种部署方式，支持将博客轻松部署到 GitHub Pages、GitLab Pages 等服务，或者部署到任意 HTTP 服务器。

由于大多数操作在命令行中完成，以及 Hexo 的静态特性，它没有后台系统。对于新手来说，可能比 Typecho 更具挑战性。开始搭建 Hexo 之前，建议先学习 Linux Shell（Windows 系统为 CMD）的基础。

安装过程中，一定要勾选“Add to PATH”，这样可以将Node.js 和npm添加到系统的环境变量中，方便在命令行中直接调用。

开始写作吧！

使用 hexo new "文章标题" 创建新文章，并在支持的 Markdown 编辑器（如 ReText）中打开并编辑。

在此推荐一款自用的Markdown编辑器，ReText。
ReText 是一款开源 Markdown 和 reStructuredText 编辑器，界面简洁，支持多平台。

写完文章后，运行 hexo g 生成页面。

Hexo 生成的静态页面可以部署到任何 HTTP 服务器，也可以使用 hexo s 搭建本地服务器或托管到 GitHub Pages 等平台。
